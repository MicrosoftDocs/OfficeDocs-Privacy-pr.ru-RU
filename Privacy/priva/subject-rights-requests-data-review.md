# <a name="review-data-for-a-subject-rights-request"></a>Проверка данных для запроса прав субъекта

После создания запроса прав субъекта [(](subject-rights-requests-create.md)дополнительные сведения) в Microsoft Priva решение "Запросы прав субъекта" будет использовать ваши входные данные для поиска совпадений с субъектом данных в среде Microsoft 365 организации. После компиляции этих данных вы можете просмотреть результаты, выбрать, что включить, и при необходимости отладить сведения. Эти действия могут совместно выполняться несколькими пользователями с помощью интерфейса Priva.

## <a name="step-1-review-request-details-and-monitor-progress"></a>Шаг 1. Просмотр сведений о запросе и отслеживание хода выполнения

Чтобы просмотреть начальные результаты поиска, перейдите в область Priva на портале соответствия [требованиям Microsoft Purview](https://compliance.microsoft.com/) и откройте **запросы прав субъекта**. Список всех открытых запросов на права субъекта можно найти на этой главной странице.

Выберите запрос в списке, чтобы просмотреть сведения о запросе. Здесь вы можете узнать больше о свойствах запроса, результатах поиска и состоянии запроса. Эта страница станет вашим центром для работы и совместной работы над найденными файлами, созданием отчетов и экспортом, а также выполнением запроса.

На **вкладке** "Обзор" страницы сведений о запросе содержатся сведения о запросе, индикатор хода выполнения, показывающий текущий шаг, и основные сведения об найденных данных. Плитки на этой странице включают в себя следующие:

##### <a name="details"></a>Details

На **карточке** "Сведения" отображаются основные сведения, чтобы ориентировать вас на запрос, например крайний срок, дата создания, описание и норматив конфиденциальности, связанные с запросом.

##### <a name="progress"></a>Ход выполнения

**Карточка хода** выполнения выводит список каждого этапа процесса: оценка данных, извлечение данных, просмотр данных, создание отчетов и закрытие запроса. Заполненный синий круг рядом с шагом указывает на текущий шаг. Флажок внутри синего круга означает, что шаг завершен, а не заполненный круг означает, что шаг еще не запущен.

##### <a name="total-number-of-items-found"></a>Общее количество найденных элементов

Статистика о текущем этапе выполнения. На этой плитке могут отображаться такие сведения, как сводка оценки данных, количество элементов, найденных в поиске и их расположения в Microsoft 365, или состояние экспорта.

##### <a name="priority-items-to-review"></a>Элементы приоритета для проверки

В **элементах приоритета** для просмотра плитки отображаются элементы, которые могут потребоваться приоритизировать при запуске проверки. На плитке отображается количество элементов, относящихся к следующим категориям:
- **Конфиденциально**. Это элементы, к ним применена метка [конфиденциальности](/microsoft-365/compliance/sensitivity-labels) Майкрософт. Например, документ Word с меткой "Строго конфиденциально". 
- **Данные с несколькими пользователями**. Эти элементы содержат персональные данные нескольких человек. Если вы хотите включить эти элементы в окончательный пакет данных, необходимо отладить ненужные данные в файлах. [Дополнительные сведения см. в разделе "Шаг 3. Просмотр](#step-3-review-data) данных ниже". Обратите внимание, что чтобы Priva определяла элементы с данными с несколькими пользователями, вашей организации необходимо настроить сопоставление данных для [запросов на права субъекта](subject-rights-requests-data-match.md).

**Как найти приоритетные элементы:**

Во-первых, убедитесь, что вы включили их представление  в таблице элементов, собранных данными, выполнив следующие действия:

- На **вкладке "Собранные данные** " выберите **"Настроить столбцы** " в верхней части списка элементов.
- Во **всплывающем окне** "Изменение столбцов" установите флажок рядом с **типами приоритетов**.
- Нажмите **Применить**. Список элементов теперь будет содержать **столбец "Типы приоритетов** ".

Теперь можно определить элементы приоритета и найти их, отсортируйте столбец типа **Priority** для группировки похожих типов.

### <a name="understand-progress-stages"></a>Общие сведения о этапах выполнения

Запросы прав субъекта проходит несколько этапов. Некоторые состояния перемещаются автоматически, а другие этапы перемещаются, когда администраторы и участники запроса прав субъекта завершат важные действия, такие как просмотр файлов.

Так как со временем или несколькими участниками может потребоваться обработка запросов, Priva постоянно обновляет состояние и рекомендации по следующим шагам. Эти обновления можно просмотреть на вкладке **"Обзор** " страницы сведений о запросе прав субъекта.

#### <a name="data-estimate"></a>Оценка данных
После создания запроса Priva немедленно начинает поиск потенциальных совпадений с субъектом данных в Microsoft 365 среде. После определения всех элементов, которые соответствуют вашим критериям, вы увидите оценку в сводной карточке оценки  данных на странице **обзора** запроса. Объем данных в области поиска повлияет на продолжительность времени, необходимого для завершения оценки.

Запрос автоматически перемещается на следующий этап извлечения данных, где все элементы содержимого собираются вместе, чтобы заинтересованные лица могли совместно работать над проверкой данных. В некоторых случаях мы приостановим оценку данных перед переходом к извлечению и уведомим вас о следующих действиях, прежде чем продолжить.

Вы также можете автоматически приостановить работу на этапе оценки данных при первом создании запроса прав субъекта. В процессе создания выберите параметр **"Получить оценку первым** " на шаге " **Параметры поиска** ". Просмотрите сведения о [шаге параметров поиска](subject-rights-requests-create.md#define-search-settings).

#### <a name="pause-in-data-estimate-for-large-search-results"></a>Приостановка оценки данных для больших результатов поиска

Priva заметит, что при оценке данных будет возвращено большое количество проверяемых элементов (более 10 000 элементов). Оценка будет приостановлена, чтобы можно было просмотреть результаты и решить, следует ли [](subject-rights-requests-create.md#refine-your-search) изменить поисковый запрос для более конкретных расположений или условий, или продолжить извлечение определенных элементов.  Мы покажем на экране количество элементов и объем данных, соответствующих вашему поиску. В строке сообщений в верхней части экрана будет один или оба следующих параметра:

- **Кнопка "Изменить поисковый** запрос" позволяет перейти непосредственно в параметры поиска запроса, чтобы задать более строгие параметры и создать новую оценку.
- Если поисковый запрос не содержит более 300 000 элементов, вы также увидите возможность **получения данных**. Это позволяет не изменять поиск и продолжать сбор данных.

#### <a name="retrieve-data"></a>Получение данных
Этап извлечения данных — это получение всех файлов, сообщений электронной почты, чатов, изображений и других элементов содержимого, содержащих персональные данные субъекта данных, и их совместное использование в контейнере хранилища BLOB-объектов Azure для проверки. Извлечение данных может занять несколько минут или значительно дольше в зависимости от объема данных. После завершения этого этапа запрос автоматически переходит к следующему этапу данных **проверки**.

#### <a name="review-data"></a>Просмотр данных
 На этом этапе участники должны просмотреть результаты на вкладке "Собранные данные" и выполнить все применимые задачи, такие как редактирование, применение тегов и добавление заметок. Завершив проверку, нажмите кнопку " **Завершить проверку"**.

#### <a name="generate-reports"></a>Создание отчетов
На этом этапе создаются отчеты. По завершении их можно найти на **вкладке "Отчеты** ". Готовые файлы можно экспортировать для окончательной проверки и доставки субъекту данных, выполнившего запрос.

#### <a name="close-the-request"></a>Закрытие запроса
Закрытый запрос указывает, что все работы по выполнению этого запроса прав субъекта завершены. Все собранные данные и отчеты будут храниться в соответствии с параметрами [хранения данных](priva-settings.md#data-retention-periods).

## <a name="step-2-optional-view-and-edit-search-queries"></a>Шаг 2 (необязательно): просмотр и изменение поисковых запросов

Чтобы просмотреть подробные сведения о поиске данных за запросом прав субъекта, выберите **"Просмотреть сведения о поисковом запросе"**. Откроется панель с суммированием запроса и отображением дополнительных сведений о том, что было найдено.

Здесь можно просмотреть результаты поиска, чтобы узнать, какой тип содержимого будет возвращен для этого запроса. Если вы хотите изменить свойства этого поиска и еще не начали этап получения данных, можно использовать параметр **"Изменить поисковый запрос** ".

Интерактивный процесс изменения поискового запроса позволяет изменять или добавлять свойства для идентификации субъекта данных, фильтров и условий поиска, а также расположений для поиска данных (включая Exchange, SharePoint, OneDrive и (или) Teams). Используйте эти параметры, чтобы достичь нужного уровня детализации. Вы можете просмотреть окончательную версию нового запроса, прежде чем нажимать кнопку **"Сохранить"**.

После редактирования поискового запроса будет выполняться новый поиск, заменять предыдущие результаты поиска. Это сбрасывает состояние в разделе **"** Ход выполнения" на первый шаг оценки **данных**. Новый поиск может занять до 60 минут. После этого вы увидите обновленные результаты на странице сведений о запросе.

## <a name="step-3-review-data"></a>Шаг 3. Просмотр данных

На этом этапе участники должны просмотреть результаты на вкладке **"Собранные данные**". Автоматически Teams канал для упрощения проверки содержимого всеми заинтересованными лицами. [Дополнительные сведения см. в статье "Совместная работа над](#collaborate-on-data-review) проверкой данных". Основные задачи для шага проверки данных описаны ниже.

#### <a name="mark-items-as-include-or-exclude-and-add-notes"></a>Пометка элементов как "Включить" или "Исключить" и "Добавить заметки"

Просмотрите список идентифицированных элементов, возвращаемых при поиске. Если вы решили, что элемент должен быть включен в окончательный отчет субъекту данных, выберите "Включить" на панели  команд в верхней части списка элементов. Вы также можете нажатие синей кнопки **"** Включить" в области проверки содержимого справа от списка элементов. При выборе **параметра "Включить**" появляется всплывающее окно с возможностью добавления заметок. Когда все будет готово, нажмите кнопку **"Отправить** ", чтобы сохранить состояние проверки элемента как **Include**.

Если элемент не входит в состав запроса, нажмите кнопку **"** Исключить" на панели команд или кнопку "Исключить" в области проверки содержимого. Исключение элемента означает, что он не будет включен в окончательные отчеты, созданные [для субъекта данных](subject-rights-requests-reports.md).

> [!NOTE]
> Если пометить элемент как **"** Исключить", необходимо добавить примечание в качестве обоснования того, почему он не относится к запросу прав субъекта. Заметки предназначены для внутренних целей и не включаются в окончательные отчеты.

Если содержимое отображается как ложное срабатывание, выберите "Не **совпадать** ", а затем на всплывающей панели нажмите кнопку **"Подтвердить"**. Это действие исключит файл из окончательных отчетов и пометит элемент как то, что не должно было быть обнаружено в поиске.

#### <a name="apply-tags"></a>Применение тегов

Теги можно использовать для идентификации элементов, которые требуют дальнейшего внимания. Priva предоставляет три тега по умолчанию — **"** Дальнейшие действия", **"** Удаление" и **"** Обновление", для которых можно задать описание. Priva также предоставляет два настраиваемых тега, которые можно использовать для имен и описания.

Например, если во время проверки данных вы определили, что элемент контента не требуется хранить в вашей организации, можно применить тег **Delete** , а затем экспортировать список всех файлов с тегами, чтобы можно было вернуться и удалить указанные элементы после выполнения запроса.

Пять тегов, управляемых **в Параметры** применяются ко всем запросам прав субъекта.

**Чтобы добавить или удалить теги:**

- Выберите элемент из списка на вкладке "Данные, **собранные** " запроса.
- В области предварительного просмотра элементов справа от списка нажмите кнопку "Применить теги" в нижней строке. Можно также выбрать три точки справа от имени элемента и выбрать параметр **"Применить теги** ".
- Появится всплывающее окно со списком тегов. Установите флажок рядом с любым тегом, который вы хотите применить к элементу. Если флажок не установлен, тег будет удален.
- Когда все будет готово, нажмите кнопку **"** Сохранить", чтобы сохранить выбранные теги и закрыть всплывающую панель.

**Чтобы добавить настраиваемые теги или обновить описания тегов:**
- На странице "Запросы прав **субъекта"** выберите Параметры в правом верхнем углу экрана, чтобы ступить к параметрам Priva.
- Перейдите на **страницу** тегов проверки данных и выберите тег, чтобы ввести описание, а для пользовательских тегов — имя. Дополнительные сведения о [параметрах тегов](priva-settings.md#data-review-tags).

**Чтобы экспортировать список помеченных элементов:**
- Перейдите на **страницу "Собранные данные** " в запросе прав субъекта.
- Над списком элементов щелкните значок стрелки вниз с надписью **"Экспорт** " при наведении на него указателя мыши.
- Скачает Excel, в котором отображаются свойства всех элементов, собранных при поиске запроса. Найдите **столбец "Теги** " для идентификации и сортировки элементов по тегу.

#### <a name="use-the-annotate-command-to-redact-text"></a>Использование команды "Заметки" для повторной обработки текста
Команда **Annotate** в области проверки содержимого позволяет создавать встроенные разметки и отлагоировать данные в элементе контента. Например, если необходимо включить файл для отдельного пользователя, который также содержит персональные данные другого субъекта данных, можно использовать изменение области под  кнопкой "Рисование" на панели команд, чтобы отключить все сведения, не относящиеся к пользователю, выполнившего запрос. Когда изменения будут завершены, выберите " **Включить** ", чтобы добавить в запрос отредактированные файлы. Заметка создает копию файла, который хранится в BLOB-объекте Azure. Исходный файл остается неизменным и хранится в исходном расположении.

#### <a name="enter-notes-about-a-file"></a>Ввод заметок о файле
Чтобы добавить или просмотреть заметки к элементу, выберите его из строки и перейдите на вкладку **"** Заметки к файлам" в области проверки содержимого справа. Вы также можете использовать параметр **"Добавить заметку** к файлу", чтобы создать комментарий. Чтобы просмотреть или добавить заметки на общем уровне обращения, перейдите на  главную вкладку "Заметки" выше и используйте примечание **"Добавить регистр"**. Эти заметки будут видны пользователям, работающим над запросом, но не будут включены в окончательный отчет или иным образом предоставлены субъекту данных.

#### <a name="complete-the-review"></a>Завершение проверки

После проверки всех элементов и установки их состояния "Включить **", "** Исключить" или "Не совпадать"**,** можно закрыть шаг проверки, настроив кнопку "Завершить проверку" в правом верхнем углу запроса. Всплывающее меню отобразит сводку данных и добавит все связанные заметки. Эти заметки предназначены для внутреннего хранения записей и не являются общими для субъекта данных.

Нажмите **кнопку "** Завершить проверку" на всплывающей панели, чтобы завершить шаг проверки. Сводки по вашим решениям будут предоставлены позже на **вкладке "Отчеты** ".

### <a name="collaborate-on-data-review"></a>Совместная работа над проверкой данных

Priva поддерживает совместную работу Microsoft Teams, чтобы ваша группа совместно работает над запросами прав субъектов. При создании нового запроса автоматически создается канал Teams, связанный с запросом по умолчанию. Здесь можно обсудить запрос и безопасно поделиться входными данными и вкладами. Чтобы присоединиться к беседе, откройте запрос и используйте параметр **"Чат с участниками совместной работы** ". Откроется Microsoft Teams на сайте группы запроса прав субъекта в канале "Общие".

Чтобы просмотреть список активных участников совместной работы, которые могут просматривать сайт группы и участвовать в его работе, в запросе прав субъекта откройте вкладку **"Участники совместной** работы". Чтобы добавить дополнительных пользователей для совместной работы над этим запросом, выберите параметр **"Добавить участника совместной работы"**.

Чтобы изменить поведение по умолчанию при создании Teams при создании запроса прав субъекта, перейдите к Параметры в  верхней области навигации и выберите Teams **совместной** работы, чтобы изменить параметр.

Вы также можете использовать параметр  "Общий доступ" в правом верхнем углу запроса субъекта, чтобы зациклить пользователей по Teams или электронной почте или скопировать ссылку на страницу в Priva. Общий доступ через Teams позволяет выбрать существующий сайт Teams и канал, доступные для вашей учетной записи, где он будет публиковать ссылку на этот случай вместе с любым сообщением, которое вы указали.

## <a name="step-4-close-the-request"></a>Шаг 4. Закрытие запроса

После выполнения всех необходимых действий для разрешения запроса прав субъекта нажмите кнопку **"Закрыть запрос"**. При этом создается окончательный отчет, который можно найти на **вкладке "Отчеты"**. Завершение может занять некоторое время в зависимости от количества файлов в запросе.
