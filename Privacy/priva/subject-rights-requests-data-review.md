# <a name="review-data-for-a-subject-rights-request"></a>Просмотр данных для запроса на права субъекта

После создания запроса на права субъекта [(](subject-rights-requests-create.md)дополнительные сведения) в Microsoft Priva решение "Запросы на права субъекта" будет использовать ваши входные данные для поисков совпадений по теме данных в Microsoft 365 среде организации. После компиляции этих данных можно просмотреть результаты, принять решение о том, что включать, и при необходимости отредактировать сведения. Эти действия могут быть совместно с несколькими пользователями с помощью интерфейса Priva.

## <a name="step-1-review-request-details-and-monitor-progress"></a>Шаг 1. Просмотр сведений о запросе и отслеживание прогресса

Чтобы увидеть начальные результаты поиска, перейдите в область Прива Центр соответствия требованиям Microsoft 365 и откройте **запросы на права субъекта**.[](https://compliance.microsoft.com/) Список всех запросов на права открытых субъектов можно найти на этой главной странице.

Выберите запрос в списке, чтобы увидеть сведения о запросе. Здесь вы можете узнать больше о свойствах запроса, результатах поиска и состоянии запроса. Эта страница станет вашим центром для работы и совместной работы по управлению найденными файлами, созданию отчетов и экспорта и выполнению запроса.

Плитки на странице сведения о запросе включают:

- **Сведения**. Основные сведения о запросе, включая его крайний срок и дату запроса, его описание и соответствующие правила конфиденциальности.
- **Прогресс**. Временная шкала, указывающая завершенные действия и все задачи, которые еще не завершены.
- Статистика текущего этапа выполнения. Эта плитка может показывать сведения, например сводку оценки данных, количество элементов, найденных в вашем поиске и их расположения в Microsoft 365, или состояние экспорта.
- **Приоритетные элементы, которые необходимо** просмотреть. На этой плитке будут показываться сведения о всех важных для вас элементов, обнаруженных в Priva. Например, это могут быть конфиденциальные сведения, уже несущие метку конфиденциальности Майкрософт, или элементы с данными о нескольких лицах, которые могут потребовать повторного анализа. Это поможет администраторам узнать, с чего начать работу в своем обзоре. Элементы приоритета можно найти в статье Data collected by filtering by the "Priority Types" column.

### <a name="understand-progress-stages"></a>Понимание этапов прогресса

Запросы на права субъекта проходят несколько этапов. Некоторые состояния автоматически прогрессируют, а другие этапы заранее, когда администраторы и участники запроса на права субъекта завершат необходимые действия, такие как просмотр файлов.

Так как запросы могут нуждаться в том, чтобы работать со временем или несколькими участниками, Priva постоянно обновляет состояние и руководство по следующим шагам. Эти обновления можно просмотреть на вкладке **Обзор** сведений о запросе на права субъекта.

#### <a name="data-estimate"></a>Оценка данных
После создания запроса Прива немедленно начинает поиск потенциальных совпадений с субъектом данных в Microsoft 365 среде. После определения всех элементов, которые соответствуют вашим критериям, вы увидите оценку в сводной карте оценки данных  на странице **Обзор** запроса. Количество данных в области поиска влияет на продолжительность времени, необходимого для завершения оценки.

Запрос будет автоматически переходить к следующему этапу сбора данных, где все элементы контента собираются вместе, чтобы заинтересованные стороны могли совместно работать над обзором данных. В некоторых случаях мы приостановим оценку данных перед переходом на поиск и уведомим вас о следующих шагах, которые необходимо предпринять перед продолжением.

Вы также можете автоматически приостановиться на этапе оценки данных при первом создании запроса на права субъекта. В процессе создания выберите параметр **Get an Estimate first** во время шага **Параметры поиска** . Просмотрите сведения о [шаге параметры поиска](subject-rights-requests-create.md#define-search-settings).

#### <a name="pause-in-data-estimate-for-large-search-results"></a>Приостановка оценки данных для больших результатов поиска

Прива заметит, если предполагается, что при оценке данных будет возвращено большое количество элементов для проверки (более 10 000 элементов). Оценка будет приостановлена, чтобы можно было просмотреть результаты и решить, изменить ли [](subject-rights-requests-create.md#refine-your-search) поисковый запрос, чтобы наметить более конкретные расположения или условия, или продолжить извлечение определенных элементов.  Мы покажем на экране количество элементов и объем данных, которые соответствуют вашему поиску. В панели сообщений в верхней части экрана будет один или оба следующих варианта:

- **Кнопка "** Изменить запрос поиска" будет непосредственно в параметры поиска запроса, чтобы установить более строгие параметры и создать новую оценку.
- До тех пор, пока запрос поиска не более 300K элементов, вы также увидите возможность **получения данных**. Это позволяет не изменять поиск и продолжать сбор данных.

#### <a name="retrieve-data"></a>Извлечение данных
Этап получения данных — это когда все файлы, электронная почта, чаты, изображения и другие элементы контента, содержащие личные данные субъекта данных, извлекаются и помещаются в контейнер для хранения blob Azure для проверки. Сбор данных может занять несколько минут или значительно больше времени в зависимости от объема данных. По завершению этого этапа запрос автоматически перемещается на следующий этап данных **Review**.

#### <a name="review-data"></a>Просмотр данных
 На этом этапе ваши участники должны просмотреть результаты в вкладке Data **collected** и выполнить все применимые задачи, такие как пересмотр, применение тегов и добавление заметок. После завершения проверки выберите полный **обзор**.

#### <a name="generate-reports"></a>Создание отчетов
На данном этапе создаются отчеты. По завершению их можно найти в вкладке **Отчеты** . Готовые файлы можно экспортировать для окончательной проверки и доставки субъекту данных, который сделал запрос.

#### <a name="close-the-request"></a>Закрыть запрос
Закрытый запрос указывает, что все работы по выполнению этого запроса прав субъекта завершены. Все собранные данные и отчеты будут храниться в соответствии с вашими [настройками хранения данных](priva-settings.md#data-retention-periods).

## <a name="step-2-optional-view-and-edit-search-queries"></a>Шаг 2 (необязательный): просмотр и изменение поисковых запросов

Чтобы просмотреть подробные сведения о поиске данных за запросом на права субъекта, выберите **сведения о запросе поиска**. Это открывает области, суммирование запроса и отображение дополнительных сведений о том, что было найдено.

Здесь можно просмотреть результаты поиска, чтобы узнать, какой тип контента будет возвращен для этого запроса. Если вы хотите изменить свойства этого поиска и еще не приступили к этапу Получения данных, вы можете использовать параметр **Edit search query** .

Управляемый процесс редактирования поисковых запросов позволяет изменять или добавлять свойства для идентификации субъекта данных, фильтров и условий поиска, а также расположения для поиска данных (в том числе Exchange, SharePoint, OneDrive и/или Teams). Используйте эти параметры, чтобы достичь нужного уровня специфичности. Вы можете просмотреть окончательную версию нового запроса перед нажатием **сохранить**.

После редактирования поискового запроса будет работать новый поиск, который заменит предыдущие результаты поиска. Это сбрасывает состояние в разделе **Прогресс** на первый шаг, **оценка данных**. Новый поиск может занять до 60 минут. После этого вы увидите обновленные результаты на странице сведений о запросе.

## <a name="step-3-review-data"></a>Шаг 3. Просмотр данных

На данном этапе ваши участники должны просмотреть результаты в вкладке **Data collected**. Автоматически Teams канал для облегчения просмотра контента всеми заинтересованными сторонами. [Дополнительные сведения см. в совместном](#collaborate-on-data-review) обзоре данных. Основные задачи для шага проверки данных описаны ниже.

#### <a name="mark-items-as-include-or-exclude-and-add-notes"></a>Пометить элементы как Включить или Исключить и добавить заметки

Просмотрите список идентифицированных элементов, возвращенных при поиске. Если вы решили, что элемент должен быть включен в итоговый отчет в субъект данных, выберите **Включить** в командную планку в верхней части списка элементов. Вы также можете выбрать синюю кнопку **Включить** в области просмотра контента справа от списка элементов. При выборе **Включить** отображается поле flyout с возможностью добавления заметок. После этого выберите **Отправить** , чтобы сохранить состояние проверки элемента как **Include**.

Если элемент не принадлежит в качестве части запроса, выберите **Исключение** в панели команд или кнопку **"** Исключить" в области просмотра контента. Исключение элемента означает, что он не будет включен в окончательные отчеты, созданные [для субъекта данных](subject-rights-requests-reports.md).

> [!NOTE]
> Если вы отмечаете элемент **как Exclude**, необходимо добавить примечание в качестве обоснования того, почему он не относится к запросу на права субъекта. Заметки относятся к внутренним целям и не включаются в окончательные отчеты.

Если содержимое кажется ложным, выберите **Не** совпадать и на области вылетов выберите **Подтвердите**. Это действие исключит файл из окончательных отчетов и пометит элемент как нечто, которое не должно было быть обнаружено в поиске.

#### <a name="apply-tags"></a>Применение тегов

Теги можно использовать для определения элементов, которые требуют дополнительного внимания. Прива предоставляет три тега по умолчанию — **Последующие**, **Удалить** и **Обновить** , для которых можно установить описание. Priva также предоставляет два настраиваемые тега, которые можно назвать и описать.

Например, если во время проверки данных определяется, что элемент контента не требуется хранить в организации, можно применить тег **Delete** , а затем экспортировать список всех помеченных файлов, чтобы можно было вернуться и удалить выявленные элементы после обработки запроса.

Пять тегов, которые вы **Параметры,** применимы к всем запросам на права субъекта.

**Добавление или удаление тегов:**

- Выберите элемент из списка на вкладке **Data collected** запроса.
- В области предварительного просмотра элемента справа от списка выберите кнопку **Применить** теги в нижней строке. Вы также можете выбрать три точки справа от имени элемента и выбрать параметр **Применить теги** .
- В списке тегов отображается поле для вылетов. Проверьте поле рядом с любыми тегами, которые необходимо применить к элементу. При отсеве проверяемого окна тег удаляется.
- После этого выберите **Сохранить**, который сохраняет выбор тегов и закрывает поле вылетов.

**Добавление настраиваемой теги или обновление описаний тегов:**
- На странице Запросы на права субъекта **выберите** Параметры в правом верхнем углу экрана, чтобы добраться до параметров Привыи.
- Перейдите на **страницу тегов** обзоров данных и выберите тег, чтобы ввести описание и, для пользовательских тегов, имя. Дополнительные новости о [параметрах тегов](priva-settings.md#data-review-tags).

**Экспорт списка помеченных элементов:**
- Перейдите на **страницу "Собранные данные** " в запросе на права субъекта.
- Над списком элементов выберите значок вниз стрелки, который говорит **Экспорт** , когда вы наведите курсор над ней.
- Загружается Excel, в котором показаны свойства для всех элементов, собранных при поиске запроса. Найдите **столбец Теги** для идентификации и сортировки элементов по тегу.

#### <a name="use-the-annotate-command-to-redact-text"></a>Чтобы отредактировать текст, используйте команду Аннотации
Команда **Аннотации** в области обзора контента позволяет создавать разметки и отредактировать данные в элементе контента. Например, если необходимо включить файл для отдельного лица, который также содержит персональные данные другого субъекта данных, можно использовать **redaction** Area под кнопкой Рисование в панели команд, чтобы отключить все сведения, не относящиеся к лицу, который сделал запрос. После завершения редактирования выберите **Включить** , чтобы добавить отредактированную папку в запрос. Аннотация создает копию файла, который хранится в вашей blob Azure. Исходный файл остается неопределенным и хранится в исходном расположении.

#### <a name="enter-notes-about-a-file"></a>Ввод записей о файле
Чтобы добавить или просмотреть заметки по элементу, выберите элемент из строки и перейдите на вкладку **Заметки** файлов в области обзора контента справа. Для создания нового комментария можно также использовать параметр **Добавить** примечание файла. Чтобы просмотреть или добавить заметки на общем уровне дела, перейдите на главную вкладку **Заметки** выше и используйте **примечание Add case**. Эти заметки будут видны пользователям, работающим с запросом, но не будут включены в окончательный отчет или иным образом делиться с субъектом данных.

#### <a name="complete-the-review"></a>Завершение проверки

Когда все элементы были рассмотрены, и вы установили их состояние Как **Включить****,** Исключить или Не совпадать **,** пришло время закрыть шаг проверки, выбрав кнопку **Полный** обзор в верхнем правом углу в запросе. На области вылетов покажут сводку данных и добавят все связанные заметки. Эти заметки для внутреннего хранения записей и не являются общими для субъекта данных.

Выберите **полный обзор** на области вылетов, чтобы завершить этап проверки. Сводки о ваших решениях будут предоставлены позже в вкладке **Отчеты** .

### <a name="collaborate-on-data-review"></a>Совместное взаимодействие в области анализа данных

Priva поддерживает совместную работу Microsoft Teams, чтобы позволить группе совместно работать над запросами на права субъектов. При создании нового запроса автоматически создается канал Teams, связанный с запросом по умолчанию. Здесь вы можете обсудить запрос и безопасно обмениваться входами и вкладами. Чтобы присоединиться к беседе, откройте запрос и используйте **параметр Чат с сотрудниками** . Это откроет Microsoft Teams и поместите вас в общий канал для сайта Команды запроса на права субъекта.

Чтобы просмотреть список активных сотрудников, которые могут просматривать и вносить свой вклад в сайт Team, в запросе на права субъекта откройте вкладку **Collaborators** . Чтобы добавить дополнительных пользователей для совместной работы по этому запросу, выберите параметр **Добавить соавтора**.

Чтобы изменить поведение по умолчанию при создании Teams сайтов при создании запроса на права субъекта, перейдите Параметры  в верхнюю Teams для изменения параметра.

Вы также можете использовать параметр **Share** в правом верхнем справа в правом запросе субъекта, чтобы зациклить людей на Teams или электронной почте или скопировать ссылку на страницу в Priva. Общий доступ Teams позволяет выбрать существующий сайт Teams и канал, доступный для вашей учетной записи, где он будет размещать ссылку на этот случай вместе с любым сообщением, которое вы поставляете.

## <a name="step-4-close-the-request"></a>Шаг 4. Закрыть запрос

Если вы выполнили все необходимые действия для разрешения запроса на права субъекта, выберите **Закрыть запрос**. Это создает окончательный отчет, который можно найти на вкладке **Отчеты**. Завершение может занять некоторое время в зависимости от количества файлов в запросе.
