# <a name="create-policies-in-privacy-risk-management"></a>Создание политик в управлении рисками конфиденциальности

Вы можете создать новые политики в службе "Управление рисками конфиденциальности" для устранения рисков, важных для вашей организации. Для быстрого запуска используйте шаблоны по умолчанию для создания новых политик для чрезмерного использования данных, передачи данных, минимизации данных и сценариев. Вы также можете настроить собственные политики, используя любой из этих шаблонов в качестве отправной точки.

При создании или изменении политик можно настроить уведомления по электронной почте или, если они доступны, подсказки политики для Teams, чтобы привлечь внимание пользователей к соответствию политикам для исправления.

## <a name="create-a-policy-from-a-template"></a>Создание политики на основе шаблона

Выполните следующие действия, чтобы создать политику с помощью любого из шаблонов по умолчанию.

1. В [Центр соответствия требованиям Microsoft 365](https://compliance.microsoft.com/) перейдите в раздел Priva Privacy Risk Management и выберите **"Политики"**.
1. Нажмите кнопку **Создать политику**.
1. Выберите нужный тип шаблона. Откроется всплывающее окно со сведениями о шаблоне.
1. Чтобы просмотреть параметры шаблона по умолчанию, включая типы данных, расположения данных и условия, которые активирует соответствие политике, выберите **"Просмотреть параметры"**.
     - Здесь можно выбрать параметр **"Изменить параметры"** для внесения изменений. Откроется мастер настройки параметров.
1. Если вы готовы использовать параметры по умолчанию, присвойте политике описательное имя и нажмите **кнопку "Создать политику".**

При создании политики непосредственно из шаблона автоматически выбирается множество параметров. Он также запускается в тестовом режиме по умолчанию. Это означает, что оповещения или уведомления не создаются. Если вы готовы полностью активировать политику после ее запуска в тестовом режиме и просмотра результатов политики, ее можно найти в списке политик и изменить политику, чтобы включить ее.

## <a name="create-a-custom-policy"></a>Создание настраиваемой политики

Чтобы получить детальный контроль над параметрами политики, можно создать настраиваемую политику, используя один из существующих шаблонов в качестве базового плана. Priva предоставляет мастер для выполнения этих действий.

Все типы политик следуют этому базовому потоку. Некоторые параметры и параметры будут изменяться в зависимости от выбранной политики.

1. В [Центр соответствия требованиям Microsoft 365](https://compliance.microsoft.com/) перейдите в раздел Priva Privacy Risk Management и выберите **"Политики"**.
1. Нажмите кнопку **Создать политику**.
1. Выберите **настраиваемый параметр** , чтобы начать работу с мастером.
1. Выберите базовый тип шаблона: **перегрузка данных,** передача **данных или** **минимизация данных**. Каждый из них предоставляет определенные параметры во время создания политики.
1. Приведите имя и опишите политику. Мы рекомендуем использовать понятные описательные имена для идентификации политик, так как эти имена будут отображаться позже в оповещениях о совпадениях политик.
1. Перейдите к мастеру и выберите нужные параметры. Варианты:
    - **Данные для мониторинга**: выберите тип персональных данных, которые будет отслеживать политика.
    - **Пользователи и группы**: примените политику ко всем пользователям или выбранным пользователям.
    - **Расположения**. Примените политику к выбранным областям в Microsoft 365.
    - **Условия**. Задайте условия для политики. Эти параметры зависят от типа политики.
    - **Результаты:** определите результаты при обнаружении соответствия политике, например уведомления пользователей.
    - **Оповещения:** определите частоту оповещений для администраторов при обнаружении соответствия политике.
    - **Режим**: сначала выберите, следует ли запускать политику в тестовом режиме.
1. Когда все параметры будут завершены, проверьте выбранные параметры, внесите необходимые изменения и нажмите **кнопку "Отправить** ", чтобы создать политику.

## <a name="learn-about-key-settings-for-all-policies"></a>Сведения о ключевых параметрах для всех политик

### <a name="choose-data-to-monitor"></a>Выбор данных для мониторинга

При изменении или настройке настраиваемой политики вам будет предложено выбрать типы данных, которые должна отслеживать политика. Доступны следующие параметры:

- **Группы классификации**: доступный для поиска список наборов данных на основе ключевых правил конфиденциальности, таких как GDPR или HIPAA. Просмотрите сведения о любой группе, чтобы узнать, какие типы конфиденциальной информации она охватывает. Выберите один или несколько из этих наборов, чтобы использовать их как есть.
- **Отдельные** типы конфиденциальной информации: самостоятельно выбрав конкретные типы конфиденциальной информации, например номера социального страхования или сведения о лицензии драйвера, вы можете настроить собственную группу или группы данных для поиска. Этот мастер позволяет выбрать из полного списка типов конфиденциальной информации в управлении рисками конфиденциальности. Каждый тип информации имеет собственные свойства. Используйте кнопку сведений рядом с любым из них, чтобы получить подробные сведения и примечания о рекомендуемых параметрах. Если вы создаете несколько групп, мастер позволяет применять логические операторы, чтобы связать их и определить их порядок операций.

Если выбрать из существующих групп классификации, вы также не сможете выбрать отдельные типы или создать собственные группы. Для максимальной гибкости выберите отдельные типы конфиденциальной информации. Чтобы использовать наиболее распространенные стандарты, выберите один из групп классификации.

### <a name="set-user-email-notifications"></a>Настройка уведомлений пользователя по электронной почте

С [помощью уведомлений по](risk-management-notifications.md) электронной почте можно отправлять уведомления о совпадениях политик непосредственно владельцам содержимого. В этих сообщениях электронной почты перечислены данные, которые необходимо проверить, и возможные действия, которые необходимо выполнить, например сделать документы закрытыми, сохранить их в файле, отправить отчеты о ложноположительных совпадениях и добавить заметки для дальнейшей справки. Эти сообщения электронной почты также содержат ссылки для обучающих получателей о том, как обрабатывать эти случаи. Предоставление этих ссылок является обязательным и должно указывать на собственную внутреннюю документацию по процессам и рекомендациям.

Уведомления можно включить для отдельных политик во время создания настраиваемой политики или при изменении любой политики. Задайте параметры в разделе **"Результаты** ".

Обязательные параметры включают частоту уведомлений и ссылку на обучение конфиденциальности.

Необязательные параметры включают определенные настраиваемые поля для сообщений электронной почты. Выберите параметр **предварительного просмотра и редактирования** сообщения уведомления, чтобы открыть всплывающую панель с примером уведомления. Здесь можно изменить тему сообщения электронной почты, заголовок и основной текст, отображаемое имя и URL-адрес для обучения конфиденциальности.

Обратите внимание, что общая возможность управления рисками конфиденциальности для отправки уведомлений по электронной почте контролируется **в Параметры**. Это приложение включено по умолчанию. Отключение этого параметра приведет к остановке всех сообщений электронной почты, даже если уведомления настроены на уровне отдельной политики.

## <a name="learn-about-settings-for-data-minimization-policies"></a>Сведения о параметрах политик минимизации данных

Политики минимизации данных ориентированы на возраст содержимого и время его последнего изменения. Мониторинг персональных данных, которые по-прежнему хранятся в устаревшем неиспользуемом содержимом, поможет вам лучше управлять хранимым данными и снизить риски. Этот параметр обрабатывается на экране **условий** .

По умолчанию политики минимизации данных искать содержимое, содержащее персональные данные, которые были созданы или изменены не менее 30 дней назад. При редактировании или создании настраиваемой политики можно выбрать один из других предустановных временных интервалов.

## <a name="learn-about-settings-for-data-transfer-policies"></a>Сведения о параметрах политик передачи данных

Политики передачи данных позволяют отслеживать передачу персональных данных за пределы организации, а также внутренние передачи между различными отделами, регионами или регионами. На экране **"Условия** " можно выбрать, какие типы передачи следует искать в службе "Управление рисками конфиденциальности".

По умолчанию политики передачи данных обнаруживают, когда персональные данные в вашей организации передаются получателю или расположению за пределами вашей организации или передаются в нее. При редактировании или создании настраиваемой политики можно выбрать тип передачи, а затем выбрать регионы или отделы отправителя и получателя.

Политики передачи данных также поддерживают предоставление советов и рекомендаций по политикам пользователям в Teams, чтобы они могли быть в курсе рекомендаций по обработке данных. Его можно включить на экране **"Результаты** ".

## <a name="learn-about-settings-for-data-overexposure-policies"></a>Сведения о параметрах политик чрезмерного использования данных

Ваша организация может хранить содержимое на различных уровнях доступа, в том числе в общедоступных и ограниченных областях. На экране **"** Условия" можно выбрать поиск потенциальной перегрузки данных для содержимого, хранимого на любом из следующих уровней доступа, в службе "Управление рисками конфиденциальности".

- **Общедоступный**: любой пользователь со ссылкой может просматривать это содержимое.
- **Внешний:** доступ есть у определенных людей за пределами организации.
- **Внутренний**: пользователи в организации имеют доступ.

По умолчанию политики перегрузки данных оценивают все три уровня доступа. При редактировании или создании настраиваемой политики можно выбрать все или любой из этих уровней.
