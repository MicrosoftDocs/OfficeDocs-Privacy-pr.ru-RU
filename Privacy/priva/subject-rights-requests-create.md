# <a name="create-a-subject-rights-request"></a>Создание запроса прав субъекта

Администраторы управления правами субъектов могут открывать новые запросы на главной странице запросов прав субъекта Priva. Мастер поможет вам найти персональные данные о субъекте данных и начать процесс выполнения запроса.

Вы также можете отправить дополнительные материалы, чтобы priva определяла субъекты данных на основе точных значений данных. Дополнительные сведения см [. в разделе "Сопоставление данных для запросов прав субъекта"](subject-rights-requests-data-match.md).

## <a name="request-types"></a>Типы запросов

Запросы прав субъекта Priva поддерживают три разных типа запросов:

1. **Access**: предоставляет сводку персональных данных субъекта данных, хранящихся в вашей организации в Microsoft 365.

2. **Экспорт**: предоставляет сводку и экспортированный файл элементов содержимого, содержащих персональные данные субъекта данных. Это элементы, которые проверяются и помечаются как включенные во время проверки данных, собранных параметрами поиска.

3. **Список с тегами для** выполнения. Создает сводку всех файлов, помеченных во время проверки данных, которые могут потребовать дополнительных действий за пределами Priva. Например, может потребоваться упростить удаление персональных данных субъекта данных в соответствии с их запросом. Вы можете просмотреть включенные теги и настроить настраиваемые теги для своей организации в [параметрах Priva](priva-settings.md).

## <a name="getting-started-with-your-first-request"></a>Начало работы с первым запросом

При запуске пробной версии или подписки на запросы прав субъекта мы предлагаем простую стандартную настройку для первого запроса, в котором используются параметры по умолчанию. Эта настройка поможет вам изучить рабочий процесс запроса прав субъекта и ознакомиться с его функциями.

При первом входе на страницу "Запросы прав субъекта" вверху отображается баннер с кнопкой **начало работы**. Когда пользователь нажмет эту кнопку, появится всплывающее окно с предварительно заполненными сведениями о нем в полях имени и электронной почты и отобразится все параметры по умолчанию.

**Изучение функциональных возможностей** запроса с информацией: пробное выполнение запроса на права субъекта на основе собственных сведений поможет вам ознакомиться с каждым этапом процесса и утешить его. Вы увидите, что по умолчанию используется для поиска, и сможете отработать уточнение результатов, настроив параметры поиска. На вкладке "Собранные данные" можно просмотреть элементы в области предварительного просмотра справа и отладить текст, применить теги, ввести заметки и пометить элементы для включения или исключения итогового отчета (сведения см. в разделе "Просмотр данных для запроса прав субъекта[").](subject-rights-requests-data-review.md)

- Вам не нужно использовать сведения для создания первого запроса. Если вы готовы начать запрос субъекта данных, просто замените свое имя и адрес электронной почты сведениями субъекта данных.

Чтобы принять все параметры и создать запрос, нажмите кнопку " **Создать"**. Область закроется, и вы увидите новый запрос, указанный на странице **"Запросы прав субъекта** ". Чтобы изменить любые параметры по умолчанию перед созданием запроса, выберите команду **"** Изменить сведения о запросе", которая помещает вас в мастер создания запроса на [права субъекта](#use-the-subject-rights-request-creation-wizard).

> [!NOTE]
> Любой запрос, который вы создаете, учитывается при использовании пробной или платной подписки независимо от того, какие сведения субъекта данных используются для запроса. Стандартный 30-дневный период хранения данных применяется после закрытия запроса. Узнайте, как изменить [периоды хранения для запросов прав субъекта](subject-rights-requests-reports.md#retention-periods-for-reports-and-data).

## <a name="use-the-subject-rights-request-creation-wizard"></a>Использование мастера создания запроса прав субъекта

1. На [портале соответствия требованиям Microsoft Purview](https://compliance.microsoft.com/) перейдите в раздел Priva и выберите "Запросы прав субъекта **Priva"**.
1. Чтобы начать новый запрос, выберите **"Создать запрос"**.
1. Определите субъект данных, отправившего запрос, и укажите его связь с вашей компанией.
1. Мы выполдим поиск элементов, связанных с субъектом данных, по умолчанию. Если вы хотите уточнить поиск или получить оценку перед получением данных, вы можете сделать эти параметры на этом этапе. Вы также можете оставить все элементы пустыми и перейти к следующему шагу. Дополнительные сведения о параметрах см. в разделе ["Определение параметров поиска](#define-search-settings) " и ["Уточнение поиска"](#refine-your-search).
1. Выберите тип запроса в зависимости от того, что субъект данных хочет сделать с их данными. Если их запрос относится к определенному нормативу конфиденциальности данных, вы также можете выбрать его из предоставленного списка, чтобы добавить дополнительный контекст. Установка крайнего срока (обязательно) упрощает сортировку при приближении или превышении запросов и своевременное их устранение. Типы запросов:
   - **Access**: предоставляет сводку персональных данных субъекта данных, хранящихся в вашей организации в Microsoft 365.
   - **Экспорт**: предоставляет сводку и экспорт персональных данных субъекта данных, собранных и аннотированных во время проверки.
   - **Список с тегами для отслеживания**: создает сводку всех файлов, которые пользователи помечаются во время проверки, которые могут потребовать дополнительных действий за пределами Priva. Например, может потребоваться упростить удаление персональных данных субъекта данных в соответствии с запросом. Пользовательскими тегами проверки данных для запросов прав субъекта можно управлять в глобальных Параметры **.**
1. Подтвердите имя этого запроса и добавьте необязательное описание для справки.
1. Просмотрите сводку того, что вы указали на предыдущих шагах. Любое поле можно изменить, прежде чем нажмем **кнопку "Создать запрос"**.

Для каждого запроса прав субъекта можно настроить поиск данных во всех или определенных расположениях в Exchange и Sharepoint. Выберите расположение, переместив переключатель в положение **"В сети** ". Вы можете выполнить поиск по всем учетным записям и сайтам или назначить определенные учетные записи или сайты в каждом расположении. Ознакомьтесь с подробными сведениями ниже о том, что рассматривается в каждом расположении.

- **Exchange**: этот параметр будет искать данные в Exchange почтовых ящиках, а также в отдельных Teams чатах. Вы можете выполнить поиск во всех учетных записях Exchange организации или выбрать "Выбрать  учетные записи", чтобы выбрать отдельных  пользователей во всплывающей Exchange почтовых ящиков.

- **SharePoint**: этот параметр будет искать данные на сайтах SharePoint, OneDrive для бизнеса сайтах и Teams каналах. Вы можете выполнить поиск по всем SharePoint сайтам в организации или выбрать "Выбрать сайты", чтобы выбрать отдельных пользователей в области  SharePoint сайтов.

Справку по выявлению соответствующих условий поиска см. в следующих разделах:

- **SharePoint** сайтов и URL-адресов: управление сайтами в центре администрирования [SharePoint](/sharepoint/manage-sites-in-new-admin-center) содержит рекомендации по сортировке и фильтрации сайтов, а также поиск SharePoint сайта. Используйте его, чтобы найти URL-адреса для ввода в поле поиска на панели **SharePoint сайтов**.

- **Teams чатов и каналов**[: Get-Team](/powershell/module/teams/get-team) показывает, как найти команды в Microsoft Teams, предоставив определенные свойства или сведения.

- **OneDrive и URL** [OneDrive](/onedrive/list-onedrive-urls#about-onedrive-urls) URL-адресах, содержат сведения о правильном формате и свойствах URL-адреса OneDrive пользователя. Используйте его для идентификации OneDrive в поиске.

Мы рекомендуем тщательно изучить выбранные параметры, чтобы определить правильную тему данных. Например, если вы выполняете поиск в почтовых ящиках по имени и находите нескольких людей с похожими именами, проверьте правильность, прежде чем добавлять их в запрос.

## <a name="define-search-settings"></a>Определение параметров поиска

При создании запроса прав субъекта поиск по умолчанию будет выполняться на основе выбранных параметров на странице  "Расположения" мастера создания. Если вы хотите выполнить более целевой поиск или получить оценку данных перед извлечением элементов контента, вы можете сделать эти параметры на странице параметров поиска мастера создания запросов.

### <a name="advanced-search-options"></a>Дополнительные параметры поиска

- **Уточнение поиска**. Этот параметр позволяет указать дополнительные свойства, которые помогут определить субъект данных в данных вашей организации. После выбора этого параметра вам будет предложено добавить дополнительные параметры поиска, как описано ниже в разделе ["Уточнение поиска"](#refine-your-search).
- **Включить контент, созданный субъектом данных**: этот параметр будет искать контент, созданный субъектом данных. Примеры включают файлы, созданные субъектом данных или SharePoint в него. Выбор этого параметра может значительно увеличить объем возвращаемых данных.
- **Включить все версии** элементов. Если вы выбрали SharePoint в качестве расположения поиска, поисковый запрос по умолчанию вернет только текущую версию SharePoint элементов. Если установлен этот флажок, будет возвращена текущая и SharePoint всех предыдущих версий элементов, что приведет к значительно большему объему данных для просмотра.

### <a name="data-estimate"></a>Оценка данных

При **выборе первого варианта "** Получить оценку" будет представлена оценка объема данных, которые мы ожидаем найти перед автоматическим извлечением данных. Когда оценка отображается на странице сведений о запросе, можно просмотреть результаты поиска и просмотреть выборку обнаруженных элементов. Если элементы представляют ожидаемые результаты, необходимо выбрать команду **"** Получить данные", чтобы продолжить фактическое извлечение элементов контента.

## <a name="refine-your-search"></a>Уточнение поиска

Если вы нажмете "Уточнить  поиск" на странице параметров поиска, при  выборе параметра "Далее" вам будет предложено указать сведения о личных атрибутах и условиях для дальнейшего выбора результатов поиска. Вы можете внести дополнения в любой или обе категории. После завершения выбора в этом разделе извлечение данных для запроса будет основано на параметрах поиска.

#### <a name="add-personal-attributes"></a>Добавление личных атрибутов

Введите значения в текстовых полях для имен, псевдонимов, адресов электронной почты и адреса субъекта данных. Можно добавить другие идентификаторы, такие как дата рождения или номер телефона, а текстовые поля поддерживают несколько записей, разделенных точкой с запятой. Когда все будет готово, нажмите **кнопку "Далее** ", чтобы перейти на **страницу условий** .

#### <a name="conditions"></a>Условия

Нажмите кнопку "Добавить условие", чтобы выбрать один из диапазонов условий для дальнейшего поиска, включая имя элемента, имена отправителей и получателей, тип персональных данных и сведения о том, был ли элемент предоставлен внешним доступом за пределами организации. Текстовые поля поддерживают несколько записей, разделенных точкой с запятой. Когда все будет готово, нажмите **кнопку "Далее** ", чтобы сохранить параметры поиска и параметров типа запроса.
