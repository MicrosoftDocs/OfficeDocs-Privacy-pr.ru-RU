# <a name="create-a-subject-rights-request"></a>Создание запроса на права субъекта

Администраторы управления правами субъектов могут открывать новые запросы на главной странице Запросы на права субъекта в Приве. Мастер поможет вам найти личные данные о субъекте данных и начать процесс выполнения их запроса.

Вы также можете загрузить дополнительный материал, чтобы приватная система определяла субъекты данных на основе точных значений предоставленных данных. Дополнительные сведения см. в [дополнительных сведениях, которые соответствуют данным для запросов на права субъекта](subject-rights-requests-data-match.md).

## <a name="request-types"></a>Типы запросов

Запросы на права субъекта Priva поддерживают три различных типа запросов:

1. **Доступ**. Предоставляет сводку персональных данных субъекта данных, удерживаемой вашей организацией в Microsoft 365.

2. **Экспорт**. Предоставляет сводку и экспортируемого файла элементов контента, содержащих персональные данные субъекта данных. Эти элементы рассматриваются и помечены как **включенные** во время просмотра данных, собранных в параметрах поиска.

3. **Помеченный список для** последующих действий. Создает сводку всех файлов, помеченных во время проверки данных, которые могут потребовать дополнительных действий за пределами Привыи. Например, может потребоваться облегчить удаление персональных данных субъекта данных в соответствии с их запросом. Вы можете просмотреть включенные теги и настроить настраиваемые теги для организации в параметрах [Priva](priva-settings.md).

## <a name="use-the-subject-rights-request-creation-wizard"></a>Использование мастера создания запросов на права субъекта

1. В [Центр соответствия требованиям Microsoft 365 выберите](https://compliance.microsoft.com/) запросы на права субъекта **Priva** из левой навигации.

2. В правом верхнем углу выберите **Создать запрос**.

3. На странице **Сведения о субъекте** данных добавьте имя и фамилию субъекта данных, адрес электронной почты, страну проживания и укажите их отношение к вашей компании. Нажмите кнопку **Далее**.

4. На странице **Расположения** выберите Microsoft 365, где необходимо искать личные данные субъекта данных. Сведения о [параметрах расположения](#choose-locations). После выбора расположения выберите **Далее.**

5. На странице **Параметры поиска** можно изменить параметры поиска, чтобы уточнить поиск и выбрать, следует ли сначала получить оценку перед поиском данных. Вам не нужно выбирать какие-либо параметры на этой странице, что приводит к поиску по умолчанию в зависимости от расположения, указанного на предыдущем шаге. Дополнительные сведения о параметрах см. в дополнительных сведениях [о параметрах поиска.](#define-search-settings) Затем выберите **Далее**.

6. На странице **Тип запроса** выберите тип запроса в зависимости от того, что субъект данных хочет сделать с их данными: **доступ****, экспорт** или список тегов для **последующей обработки**.[](#request-types) Если их запрос относится к определенному правилу конфиденциальности данных, вы также можете выбрать его из предоставленного списка, чтобы добавить дополнительный контекст. Установка крайних сроков (необходимых) упрощает сортировку приближающихся или просроченных запросов и своевременное их устранение. После этого выберите **Далее**.

7. В имени **Запрос подтвердим** или отредактировать имя для этого запроса и добавьте дополнительное описание для ссылки. Нажмите кнопку **Далее**.

8. На странице **Обзор и готовое** просмотреть выбор. Любой раздел можно изменить. По завершению выберите **создать запрос**.

Запрос будет иметь собственную страницу сведений и будет указан на главной странице запроса на права субъекта. После создания запроса Прива начнет поиск совпадений с данными субъекта данных, предоставленными в указанных вами местах. Может занять несколько минут, чтобы показать оценку данных на странице сведений запроса в зависимости от области поиска.

## <a name="choose-locations"></a>Выбор расположения

Для каждого запроса на права субъекта можно настроить поиск для поиска данных во всех или определенных расположениях в Exchange и Sharepoint. Выберите расположение, перемещая переключатель на **положение On** . Вы можете выбрать поиск всех учетных записей и сайтов или назначить определенные учетные записи или сайты в каждом расположении. Ознакомьтесь с подробными сведениями о том, что покрывается в каждом расположении.

- **Exchange**. Этот параметр будет искать данные в Exchange почтовых ящиках, а также в отдельных Teams чатах. Вы можете выбрать поиск Exchange учетных записей в организации или выбрать учетные записи  Выберите для выбора отдельных пользователей из области Exchange почтовых ящиков.

- **SharePoint**. Этот параметр будет искать данные в SharePoint сайтах, OneDrive для бизнеса сайтах и Teams каналах. Вы можете выбрать поиск SharePoint сайтов в организации или выбрать выберите сайты для выбора отдельных пользователей  из области SharePoint сайтов.

Для справки по выявлению соответствующих терминов поиска обратитесь к следующим темам:

- **SharePoint** сайтов и [URL-адресов](/sharepoint/manage-sites-in-new-admin-center). Управление сайтами в центре администрирования SharePoint дает рекомендации по сортировке и фильтрации сайтов и поиску SharePoint сайта. Используйте это, чтобы найти URL-адреса для ввода в поле поиска на SharePoint **области** вылетов сайтов.

- **Teams чатах и каналах**: [Get-Team](/powershell/module/teams/get-team) показывает, как найти группы в Microsoft Teams, предоставляя определенные свойства или сведения.

- **OneDrive** сайтов и [URL-адресов](/onedrive/list-onedrive-urls#about-onedrive-urls): OneDrive URL-адреса предоставляют сведения о надлежащем формате и свойствах для URL OneDrive пользователя. Используйте это, чтобы помочь определить OneDrive сайтов в поиске.

Мы рекомендуем тщательно проанализировать выбор, чтобы убедиться, что вы определяете правильный субъект данных. Например, при поиске почтовых ящиков по имени и поиске нескольких людей с похожими именами убедитесь в правильности, прежде чем добавлять их в запрос.

## <a name="define-search-settings"></a>Определение параметров поиска

При создании запроса на права субъекта поиск по умолчанию будет запускаться на основе ваших выборов на странице **Расположения** мастера создания. Если вы хотите провести более целенаправленный поиск или получить оценку данных до получения элементов контента, вы можете сделать эти выборы на странице Параметры поиска мастера создания запроса.

### <a name="advanced-search-options"></a>Расширенные параметры поиска

- **Уточнение** поиска. Этот параметр позволяет указать дополнительные свойства, которые помогут определить субъект данных в данных организации. После выбора этого параметра вам будет предложено добавить дополнительные параметры поиска, поясняется ниже в [уточнении поиска](#refine-your-search).
- **Включай контент,** автором который является субъект данных: этот параметр будет искать контент, который был автором субъекта данных. Примеры включают файлы, созданные или загруженные SharePoint в объект данных. Выбор этого параметра может значительно увеличить объем возвращаемой информации.
- **Включайте** все версии элементов. Если вы выбрали SharePoint в качестве расположения поиска, поисковый запрос по умолчанию возвращает только текущую версию элементов SharePoint. Проверка этого окна возвращает текущие и все предыдущие версии элементов SharePoint, что позволит вам просмотреть значительно больший объем данных.

### <a name="data-estimate"></a>Оценка данных

В **первом варианте Get an estimate** будет представлена оценка того, сколько данных мы ожидаем найти перед автоматическим извлечением данных. Когда оценка отображается на странице сведений запроса, вы можете просмотреть результаты поиска и просмотреть выборку обнаруженных элементов. Если элементы представляют ожидаемые результаты, необходимо выбрать **извлечение** данных, чтобы приступить к фактическому извлечению элементов контента.

## <a name="refine-your-search"></a>Уточнение поиска

Если **вы решите** уточнить поиск на странице Параметры поиска, при выборе **Далее** вам будет предложено предоставить сведения о личных атрибутах и условиях для дальнейшей ориентации результатов поиска. Вы можете внести дополнения в обеих категориях. После выбора в этом разделе поиск данных для запроса будет основываться на параметрах поиска.

#### <a name="add-personal-attributes"></a>Добавление личных атрибутов

Введите значения в текстовых полях для имен, псевдонимов, адресов электронной почты и имен субъекта данных. Можно добавить другие идентификаторы, такие как дата рождения или номер телефона, а текстовые поля поддерживают несколько записей, разделенных полуколоном. Когда вы закончили, выберите **Далее** , чтобы продолжить страницу **Условия** .

#### <a name="conditions"></a>Условия

Выберите кнопку Добавить условие, чтобы выбрать среди ряда условий для дальнейшего целевого поиска, включая имя элемента, имена отправитель и получатель, тип персональных данных, а также то, был ли элемент общим внешне за пределами организации. Текстовые поля поддерживают несколько записей, разделенных полуколоном. Когда вы закончили, выберите **Далее** , чтобы сохранить параметры поиска и добиться прогресса в параметре типа запроса.

## <a name="next-steps"></a>Дальнейшие действия

После создания запроса вы увидите его на странице запроса на права субъекта. Дополнительные сведения о том, как перейти к проверке, см. в [обзоре данных и совместной работе по запросам](subject-rights-requests-data-review.md).
