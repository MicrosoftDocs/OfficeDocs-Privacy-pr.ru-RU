# <a name="create-a-request-and-define-search-settings"></a>Создание запроса и определение параметров поиска

**Из этой статьи** вы узнаете, как создать запрос в Priva, чтобы начать выполнение запроса прав субъекта. Сведения о параметрах поиска для извлечения данных и о том, как уточнить поиск.

Пользователи, которые находятся в группе ролей "Администраторы запроса прав субъекта", отличной от роли "Только просмотр" (см. Priva [разрешения), могут](priva-permissions.md) создать запрос. Существует два способа создания запроса:

- В [шаблоне используется](#quick-setup-use-a-template-with-default-settings) быстрый вариант "вне сети", использующий настраиваемые параметры по умолчанию. Или
- [Пользовательский](#custom-setup-guided-process-to-choose-all-settings) параметр, который представляет собой интерактивный процесс для пошагового выполнения всех параметров.

> [!TIP]
> При первом посещении решения **"Запросы** прав субъекта" мы предлагаем упрощенный способ создания запроса и изучения функций. См [. раздел "Начало работы с первым запросом"](#getting-started-with-your-first-request) ниже.

## <a name="request-types"></a>Типы запросов

Запросы субъектов данных Priva поддерживает три разных типа запросов:

1. **Access**: предоставляет сводку персональных данных субъекта данных, хранящихся в вашей организации в Microsoft 365.

2. **Экспорт**: предоставляет сводку и экспортированный файл элементов содержимого, содержащих персональные данные субъекта данных. Это элементы, которые проверяются и помечаются как включенные во время проверки данных, собранных параметрами поиска.

3. **Список с тегами** для выполнения. Создает сводку по файлам, помеченным во время проверки данных, которые могут потребовать дополнительных действий за пределами Priva. Например, может потребоваться упростить удаление персональных данных субъекта данных в соответствии с их запросом. Вы можете просмотреть включенные теги и настроить настраиваемые теги для организации в [Priva параметрах](priva-settings.md).

## <a name="getting-started-with-your-first-request"></a>Начало работы с первым запросом

При запуске пробной версии или подписки на запросы прав субъекта мы предлагаем простую стандартную настройку для первого запроса, в котором используются параметры по умолчанию. Эта настройка поможет вам изучить рабочий процесс запроса прав субъекта и ознакомиться с его функциями.

При первом входе на страницу "Запросы прав субъекта" вверху отображается баннер с кнопкой **начало работы**. Когда пользователь нажмет эту кнопку, появится всплывающее окно с предварительно заполненными сведениями о нем в полях имени и электронной почты и отобразится все параметры по умолчанию.

**Изучение функциональных возможностей** запроса с информацией: пробное выполнение запроса на права субъекта на основе собственных сведений поможет вам ознакомиться с каждым этапом процесса и утешить его. Вы увидите, что будет возвращать поиск по умолчанию, и сможете отработать уточнение результатов, настроив параметры поиска. На вкладке "Собранные данные" можно просмотреть элементы в области предварительного просмотра справа и отладить текст, применить теги, ввести заметки и пометить элементы для включения или исключения итогового отчета (сведения см. в разделе "Просмотр данных для запроса прав субъекта[").](subject-rights-requests-data-review.md)

- Вам не нужно использовать сведения для создания первого запроса. Если вы готовы начать запрос субъекта данных, замените имя и адрес электронной почты сведениями субъекта данных.

Чтобы принять все параметры и создать запрос, нажмите кнопку " **Создать"**. Область закроется, и вы увидите новый запрос, указанный на странице **"Запросы прав субъекта** ". Чтобы изменить любые параметры по умолчанию перед созданием запроса, выберите команду **"** Изменить сведения о запросе", которая помещает вас в мастер создания запроса на [права субъекта](#custom-setup-guided-process-to-choose-all-settings).

> [!NOTE]
> Любой запрос, который вы создаете, учитывается при использовании пробной или платной подписки независимо от того, какие сведения субъекта данных используются для запроса. Стандартный 30-дневный период хранения данных применяется после закрытия запроса. Узнайте, как изменить [периоды хранения для запросов прав субъекта](subject-rights-requests-reports.md#retention-periods-for-reports-and-data).

## <a name="quick-setup-use-a-template-with-default-settings"></a>Быстрая настройка: использование шаблона с параметрами по умолчанию

При создании запроса непосредственно из шаблона параметры по умолчанию помогают быстро приступить к работе. Три шаблона соответствуют трем типам [запросов: доступ](#request-types) к **данным, экспорт** данных и **данные с** тегами **для дальнейших действий.**  Вы можете просмотреть параметры шаблона по умолчанию и внести изменения в процессе создания запроса.

##### <a name="selecting-the-relationship"></a>Выбор связи

Каждый шаблон позволяет выбрать тип связи между субъектом данных и вашей организацией, которая, в свою очередь, определяет параметры по умолчанию. В приведенных ниже пунктах объясняется, как выбранная связь влияет на параметры поиска в шаблонах:

- **Текущий** сотрудник: результаты поиска предназначены для предотвращения сообщений электронной почты или Teams чатов, в которых участвовал сотрудник, или файлов, созданных сотрудником. Этот параметр упрощает результаты поиска, так как текущие сотрудники обычно имеют доступ к содержимому, которое они создали, или обмену данными, в которых они участвовали.

- **Экс-сотрудник**: параметры поиска приоритизют почтовый ящик бывшего сотрудника (если он доступен) и элементы, созданные сотрудником. Эти параметры предназначены для возврата содержимого, сгенерированных или участвующих в них бывшего сотрудника.

- Клиент, **потенциальный** сотрудник и **другие: результаты** поиска для этих связей не будут содержать контент, созданный субъектом данных, и будут включать только последние версии элементов SharePoint (дополнительные сведения см. в дополнительных параметрах [поиска).](#advanced-search-options)

> [!TIP]
> На главной **странице "Запросы прав субъекта** " можно отсортировать список запросов по связи в столбце **"Отношение к организации** ".

#### <a name="steps-for-creating-a-request-from-a-template"></a>Действия по созданию запроса из шаблона

1. В центре [Microsoft Purview соответствия](https://compliance.microsoft.com/) требованиям **выберите Запросы субъектов данных Priva в** области навигации слева.

2. В правом верхнем углу экрана выберите " **Создать запрос"**.

3. Найдите тип запроса, который вы хотите создать — доступ к **данным, экспорт** данных или **данные с** тегами для дальнейших действий **— и выберите начало работы**. Появится всплывающее окно.

4. В **разделе "Связь с** организацией" выберите параметр, описывающий [](#selecting-the-relationship) связь между субъектом данных и вашей организацией. Если выбрать "**Другое"****, во** всплывающей области появится текстовое поле "Другое отношение". Введите термин, который подходит для вашей организации. Например, "patient" или "student". 

5. Если вы хотите просмотреть все параметры по умолчанию и внести изменения в любой из них, нажмите кнопку " **Просмотреть параметры** ". Параметры будут отображаться во всплывающей области. Чтобы изменить любые параметры, выберите пункт **"Изменить параметры**", который откроет интерактивный процесс [, описанный](#custom-setup-guided-process-to-choose-all-settings) ниже.

6. Введите сведения о субъекте данных:
    - Поля "Имя" и "Фамилия" и "Адрес электронной почты" являются обязательными. Для текущих и прежних сотрудников при вводе имени субъекта данных  в поле "Поиск субъекта данных" откроется список пользователей для выбора. Вы также можете выбрать ссылку под этим полем, чтобы вручную ввести имя и адрес электронной почты.
    - Необязательно указывать страну проживания субъекта данных, норматив конфиденциальности, связанный с запросом, и сроки.
   
    > [!TIP]
    >По умолчанию будет выполняться поиск контента, созданного или измененного в течение последних 12 месяцев. Вы можете изменить этот параметр, чтобы выбрать все содержимое, созданное в любое время, или задать настраиваемый интервал времени.

7. Когда все будет готово, нажмите кнопку **"Создать"**. Вы будете перенаправлены на страницу запроса **прав** субъекта с новым запросом, указанным в верхней части списка запросов.

По умолчанию ваш запрос называется именем субъекта данных и типом запроса. Чтобы изменить имя запроса, выберите его в списке, чтобы открыть страницу сведений, и нажмите команду **"** Изменить" в верхней части экрана. Вы получите мастер создания запросов. Нажмите **кнопку "** Далее", пока не перейдите на страницу имени запроса, где можно изменить имя и добавить описание.

Ни один из шаблонов не [приостанавливаться автоматически на этапе оценки данных](subject-rights-requests-data-retrieval.md#pause-in-data-estimate). Однако вы можете изменить этот и другие параметры поиска, выбрав Параметры на  всплывающей панели шаблона.[](subject-rights-requests-create.md#defining-search-settings)

## <a name="custom-setup-guided-process-to-choose-all-settings"></a>Пользовательская настройка: интерактивный процесс выбора всех параметров

Параметр пользовательского запроса — это интерактивный процесс создания политики. Сначала нужно выбрать шаблон, а затем просмотреть каждый параметр, чтобы настроить политику. В приведенных ниже инструкциях содержатся сведения об основных параметрах, которые применяются к каждому из трех типов политик. Если параметры различаются по типу политики, мы будем ссылались на определенные инструкции.

Выполните следующие действия, чтобы создать запрос:

1. В [Портал соответствия требованиям Microsoft Purview выберите](https://compliance.microsoft.com/) **Запросы субъектов данных Priva в** области навигации слева.

2. В правом верхнем углу экрана выберите " **Создать запрос"**.

3. В разделе **"Пользовательский**" выберите **начало работы**. Вы будете перенаправлены в мастер создания запросов.

4. На странице **сведений о субъекте** данных введите имя и фамилию субъекта данных, а также адрес электронной почты. Выберите **"Добавить место расположения** ", чтобы выбрать страну проживания субъекта данных. Выберите параметр, определяющий, как субъект данных связан с вашей организацией. Если вы **выберете "Другое**", введите термин, описывающий отношение, которое имеет смысл для вашей организации. Например, "patient" или "student". Нажмите **кнопку "** Далее", чтобы перейти к следующему шагу.

5. На странице **"** Расположения" определите, где искать сведения о субъекте данных. Выберите одно или оба из следующих расположений, переместив переключатель состояния рядом с каждым параметром в **положение "В сети** ":

   - **Exchange**: поиск данных в Exchange почтовых ящиках, а также в отдельных или групповых Teams чатах. Вы можете выполнить поиск во всех учетных записях Exchange организации или выбрать "Выбрать  учетные записи", чтобы выбрать отдельных  пользователей во всплывающей Exchange почтовых ящиков.

   - **SharePoint**: поиск данных на сайтах SharePoint, OneDrive для бизнеса сайтах и Teams каналах. Вы можете выполнить поиск по всем SharePoint сайтам в организации или выбрать "Выбрать сайты", чтобы выбрать отдельных пользователей в области  SharePoint сайтов.

    > [!TIP]
    > Справку по выявлению соответствующих условий поиска см. в следующих разделах:
    > - SharePoint сайтов и URL-адресов: управление сайтами в центре администрирования [SharePoint](/sharepoint/manage-sites-in-new-admin-center) содержит рекомендации по сортировке и фильтрации сайтов, а также по поиску SharePoint сайта. Используйте его, чтобы найти URL-адреса для ввода в поле поиска на панели **SharePoint сайтов**.
    > - Teams чатов и каналов[: Get-Team](/powershell/module/teams/get-team) показывает, как найти команды в Microsoft Teams, предоставив определенные свойства или сведения.
    > - OneDrive и [URL-адресах: url-адреса](/onedrive/list-onedrive-urls#about-onedrive-urls) OneDrive содержат сведения о правильном формате и свойствах URL-адреса OneDrive пользователя. Используйте его для идентификации OneDrive в поиске.

6. На странице **"Определение параметров поиска** " можно внести изменения в поиск по умолчанию, выбрав один из вариантов расширенного поиска. Здесь также можно выбрать получение оценки перед автоматическим возвратом данных. При выборе любого из этих параметров при выборе команды **"** Далее" вы переходите к дополнительным экранам. Дополнительные сведения см [. в разделе "Определение параметров поиска"](#defining-search-settings) ниже. Если вы не хотите изменять поиск, оставьте все параметры пустыми и нажмите кнопку **"** Далее", чтобы переход к следующему этапу.

7. На странице **"Выбор типа** запроса" выберите тип запроса **: "** Доступ **", "** Экспорт" или "Список с тегами **"** (см. [описания выше](#request-types)). Укажите, относится ли запрос к нормативу конфиденциальности данных. Просмотрите или внесите изменения в крайний срок завершения, который по умолчанию равен двум неделям после даты создания запроса. Нажмите кнопку **Далее**.

8. На странице **"Подтверждение или изменение** имени запроса" можно сохранить или изменить понятное имя, предоставленное для запроса, и ввести необязательное описание. Нажмите кнопку **Далее**.

9. На странице **"Проверка и завершение** " просмотрите сводку того, что вы указали на предыдущих шагах. Любое поле можно изменить, выбрав ссылку "Изменить **"** в каждом разделе. Когда все будет готово, нажмите кнопку " **Создать запрос"**.

После создания запроса появится экран подтверждения. Нажмите **кнопку "** Готово", чтобы вернуться на главный экран "Запросы прав субъекта". Новый запрос будет указан в верхней части списка запросов. Выберите его из списка, чтобы начать проверку и работу на этапах выполнения.

#### <a name="what-happens-after-your-request-is-created"></a>Что происходит после создания запроса

Мы вычислим оценку объема данных, которые он ожидает найти, на основе параметров поиска. Перейдите на страницу сведений о запросе за несколько минут до одного часа, чтобы просмотреть результаты оценки и узнать, перемещен ли запрос на следующий этап. Запрос будет автоматически выполняться **от оценки** данных к  получению данных, за исключением следующих случаев:

 - Если вы выбрали получение esimtae данных первым в шаге **5,** определите параметры поиска, вы сможете просмотреть сведения о поиске и внести изменения в поиск перед получением всех данных. Если вы приостановите оценку данных, вам потребуется вручную переход к следующему этапу извлечения **данных**.
 
 - Если вы не решили остановиться на оценке данных, но мы прогнозируем, что при поиске будет получен большой объем данных, вы увидите панель сообщений в верхней части запроса со ссылкой для редактирования поиска, прежде чем перейти к извлечению **данных.**

[Дополнительные сведения об этих](subject-rights-requests-data-retrieval.md) этапах см. на странице оценки и извлечения данных.

## <a name="defining-search-settings"></a>Определение параметров поиска

При создании запроса прав субъекта поиск по умолчанию будет выполняться на основе выбранных параметров на странице  "Расположения" мастера создания. Если вы хотите выполнить более целевой поиск или получить оценку данных перед извлечением элементов контента, вы можете сделать эти параметры на странице параметров поиска мастера создания запросов.

### <a name="advanced-search-options"></a>Дополнительные параметры поиска

- **Уточнение поиска**. Этот параметр позволяет указать дополнительные свойства, которые помогут определить субъект данных в данных вашей организации. После выбора этого параметра вам будет предложено добавить дополнительные параметры поиска, как описано ниже в [разделе "Уточнение поиска"](#refining-your-search).
- **Включить контент, созданный субъектом данных**: этот параметр будет искать контент, созданный субъектом данных. Примеры включают файлы, созданные субъектом данных или SharePoint в него. Выбор этого параметра может значительно увеличить объем возвращаемых данных.
- **Включить все версии** элементов. Если вы выбрали SharePoint в качестве расположения поиска, поисковый запрос по умолчанию вернет только текущую версию SharePoint элементов. Если установлен этот флажок, будет возвращена текущая и SharePoint всех предыдущих версий элементов, что приведет к значительно большему объему данных для просмотра.

> [!TIP]
> Вы можете отправить дополнительные материалы, чтобы Priva определять субъекты данных на основе точных значений данных. Дополнительные сведения см [. в разделе "Сопоставление данных для запросов прав субъекта"](subject-rights-requests-data-match.md).

### <a name="data-estimate"></a>Оценка данных

При **выборе первого варианта "** Получить оценку" будет представлена оценка объема данных, которые мы ожидаем найти перед автоматическим извлечением данных. Когда оценка отображается на странице сведений о запросе, можно просмотреть результаты поиска и просмотреть выборку обнаруженных элементов. Если элементы представляют ожидаемые результаты, необходимо выбрать команду **"** Получить данные", чтобы продолжить фактическое извлечение элементов контента. Получите дополнительные сведения о [этапе оценки данных](subject-rights-requests-data-retrieval.md).

## <a name="refining-your-search"></a>Уточнение поиска

Если выбрать "Уточнить поиск" на странице  параметров поиска или приостановить на этапе оценки данных и изменить поисковый запрос, вам будет предложено указать сведения о личных атрибутах и условиях для дальнейшей настройки результатов поиска. Вы можете внести дополнения в любой или обе категории. После завершения выбора в этом разделе извлечение данных для запроса будет основано на параметрах поиска.

Приведенные ниже сведения также будут предложено предоставить, если вы приостановите работу на этапе оценки  данных и решите изменить поисковый запрос. 

#### <a name="add-personal-attributes"></a>Добавление личных атрибутов

Введите значения в текстовых полях для имен, псевдонимов, адресов электронной почты и адреса субъекта данных. Можно добавить другие идентификаторы, такие как дата рождения или номер телефона, а текстовые поля поддерживают несколько записей, разделенных точкой с запятой. Когда все будет готово, нажмите **кнопку "Далее** ", чтобы перейти на **страницу условий** .

#### <a name="conditions"></a>Условия

Нажмите кнопку "Добавить условие", чтобы выбрать один из диапазонов условий для дальнейшего поиска, включая имя элемента, имена отправителей и получателей, тип персональных данных и сведения о том, был ли элемент предоставлен внешним доступом за пределами организации. Текстовые поля поддерживают несколько записей, разделенных точкой с запятой. Когда все будет готово, нажмите **кнопку "Далее** ", чтобы сохранить параметры поиска и параметров типа запроса.
